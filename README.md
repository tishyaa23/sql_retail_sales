# 🛍️ Retail Sales Analysis using PostgreSQL

## 📖 Project Overview

This project demonstrates an end-to-end retail sales analysis workflow using **PostgreSQL**. The objective was to transform raw retail transaction data into meaningful business insights by performing database creation, data cleaning, exploratory data analysis (EDA), and SQL-based business analysis.

The project follows a structured data analytics pipeline, beginning with database design and data validation, followed by solving real-world business problems using SQL queries.

---

## 🎯 Objectives

- Design a PostgreSQL database for retail sales data.
- Import and organize transaction data from an Excel dataset.
- Perform data cleaning and validation to ensure data quality.
- Conduct exploratory data analysis (EDA).
- Solve business-related analytical questions using SQL.
- Generate meaningful insights from retail sales data.

---

## 📂 Dataset Information

The dataset contains retail transaction records with the following attributes:

| Column | Description |
|---------|-------------|
| transaction_id | Unique transaction identifier |
| sale_date | Date of purchase |
| sale_time | Time of purchase |
| customer_id | Customer identifier |
| gender | Customer gender |
| age | Customer age |
| category | Product category |
| quantity | Number of items purchased |
| price_per_unit | Price of one unit |
| cogs | Cost of goods sold |
| total_sale | Total transaction amount |

# 🧹 Data Cleaning & Exploratory Data Analysis (EDA)

Before performing any business analysis, several data quality checks were carried out to ensure the dataset was clean and reliable.

### Data Cleaning Steps

- Verified the uniqueness of Transaction IDs.
- Checked for duplicate records.
- Identified and removed NULL values.
- Validated the format of the `sale_date` column.
- Examined product categories for inconsistencies.
- Verified quantity values.
- Performed a final data quality check before analysis.

These preprocessing steps helped improve the accuracy and reliability of the subsequent SQL analysis.

# 📊 Business Problems Solved

The following business questions were answered using SQL:

### 1. Retrieve all sales made on **5th November 2022**.

### 2. Retrieve all Clothing transactions where:
- Quantity sold was greater than 10
- Sale occurred during November 2022

### 3. Calculate the total sales generated by each product category.

### 4. Find the average age of customers purchasing Beauty products.

### 5. Retrieve all transactions where total sales exceeded **1000**.

### 6. Calculate the total number of transactions made by each gender within every category.

### 7. Calculate the average monthly sales and identify the best-selling month for each year.

### 8. Find the Top 5 customers based on total purchase value.

### 9. Calculate the number of unique customers for each product category.

### 10. Categorize transactions into Morning, Afternoon, and Evening shifts and calculate the number of orders in each shift.
